To be really precise, the “number” type can store larger integers (up to 1.7976931348623157 * 10308),
but outside of the safe integer range ±(253-1) there will be a precision error
because not all digits fit into the fixed 64-bit storage.
So an “approximate” value may be stored. example: console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992

BigInt type was recently added to the language to represent integers of arbitrary length.
